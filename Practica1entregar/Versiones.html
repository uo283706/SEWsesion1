RUBY

Principales versiones y sus respectivos cambios en Ruby:

Ruby 1.8

Se lanzó inicialmente en agosto de 2003, se mantuvo estable durante mucho tiempo y se retiró en junio de 2013.

Ha sido objeto de varios estándares de la industria. Las especificaciones de idioma para Ruby fueron desarrolladas por el Centro de Promoción de Estándares Abiertos de la Agencia de Promoción de Tecnologías de la Información para su presentación al Comité de Normas Industriales de Japón (JISC) y luego a la Organización Internacional de Normalización (ISO). Fue aceptado como norma industrial japonesa (JIS X 3017) en 2011 y como norma internacional ( ISO / IEC 30170 ) en 2012.

Alrededor de 2005, el interés en el lenguaje Ruby surgió junto con Ruby on Rails , un marco web escrito en Ruby. Con frecuencia se atribuye a Rails el aumento de la conciencia de Ruby.

Ruby 1.9

Fue lanzado el día de Navidad de 2007. Ruby pasó de tener una licencia doble bajo la Licencia Ruby y la GPL a tener una licencia doble bajo la Licencia Ruby y la licencia BSD de dos cláusulas.

La adopción de 1.9 fue ralentizada por cambios de 1.8 que requirieron la reescritura de muchas gemas populares de terceros .

Ruby 1.9 introduce muchos cambios significativos sobre la serie 1.8:

  1. Bloquear variables locales (variables que son locales al bloque en el que se declaran)
  2. Una sintaxis lambda adicional :f = ->(a,b) { puts a + b }
  3. Una sintaxis literal de Hash adicional usando dos puntos para las claves de símbolo:{symbol_key: "value"} == {:symbol_key => "value"}
  4. Se admiten codificaciones de caracteres por cadena
  5. Nueva API de socket ( soporte IPv6 )
  6. Require relativa seguridad de importación

Ruby 2.0

Ruby 2.0 estaba destinado a ser totalmente compatible con versiones anteriores de Ruby 1.9.3. A partir del lanzamiento oficial 2.0.0 del 24 de febrero de 2013, solo había cinco incompatibilidades conocidas (menores).

Ruby 2.0 agregó varias características nuevas, que incluyen:
  1. Argumentos de la palabra clave del método.
  2. Un nuevo método, Module#prepend, para extender una clase
  3. Un nuevo literal para crear una matriz de símbolos
  4. Nueva API para la evaluación diferida de Enumerables
  5. Una nueva convención de usar #to_h para convertir objetos a Hashes.

Ruby 2.1

Se lanzó el día de Navidad de 2013. El lanzamiento incluye aceleraciones, correcciones de errores y actualizaciones de la biblioteca.

La política de control de versiones de Ruby se parece más al control de versiones semántico.

Aunque similar, la política de versiones de Ruby no es compatible con las versiones semánticas:

Ruby Control de versiones semántico
MAYOR Control de versiones semántico
MAYOR Aumentado cuando el cambio es incompatible que no se puede lanzar en MENOR. Reservado para eventos especiales
MENOR Aumenta cada Navidad, puede ser incompatible con API
MENOR Aumenta cuando agrega funcionalidad de una manera compatible con versiones anteriores
TEENY Seguridad o corrección de errores que mantiene la compatibilidad con API. Puede aumentarse más de 10 (como 2.1.11) y se publicará cada 2-3 meses
PARCHE Aumenta cuando realiza correcciones de errores compatibles con versiones anteriores
PARCHE Número de confirmaciones desde la última versión MINOR (se restablecerá a 0 cuando se libere MINOR)

Ruby 2.2

Se lanzó el día de Navidad de 2014. La versión incluye aceleraciones, correcciones de errores y actualizaciones de bibliotecas y elimina algunas API obsoletas.

En particular, introduce cambios en el manejo de la memoria: un recolector de basura incremental, soporte para la recolección de basura de símbolos y la opción de compilar directamente contra jemalloc.

También contiene soporte experimental para usar vfork (2) con system () y spawn (), y soporte adicional para la especificación Unicode 7.0.

Las características que se hicieron obsoletas o se eliminaron incluyen callcc, la biblioteca DL, Digest :: HMAC, lib / racional.rb, lib / complex.rb, GServer, Logger :: Application, así como varias funciones de la API de C.

Ruby 2.3

Ruby 2.3.0 se lanzó el día de Navidad de 2015. Algunos cambios notables incluyen:

  1. La capacidad de marcar todos los literales de cadena como congelados de forma predeterminada con un gran aumento del rendimiento en las operaciones de cadena.
  2. Comparación de hash para permitir la verificación directa de pares clave / valor en lugar de solo claves.
  3. Un nuevo operador de navegación seguro &. que puede facilitar el manejo nulo (por ejemplo, en lugar de , podemos usar ).if obj && obj.foo && obj.foo.barif obj&.foo&.bar
  4. La gema did_you_mean ahora se incluye de forma predeterminada y se requiere en el inicio para sugerir automáticamente coincidencias de nombres similares en un NameError o NoMethodError .
  5. Hash # dig y Array # dig para extraer fácilmente valores profundamente anidados (por ejemplo , dado , el valor Foo Baz ahora se puede recuperar ).profile = { social: { wikipedia: { name: 'Foo Baz' } } }profile.dig(:social, :wikipedia, :name)
  6. .grep_v(regexp) que coincidirá con todos los ejemplos negativos de una expresión regular dada, además de otras características nuevas.

La rama 2.3 también incluye muchas mejoras de rendimiento, actualizaciones y correcciones de errores, incluidos cambios en la llamada Proc #, uso de Socket y IO de palabras clave de excepción, manejo de nombres de Thread #, conexiones Net :: FTP pasivas predeterminadas y Rake eliminado de stdlib.

Ruby 2.4

Ruby 2.4.0 se lanzó el día de Navidad de 2016.

Algunos cambios notables incluyen:

  1. Binding # irb: Inicie una sesión REPL similar a binding.pry
  2. Unificar Fixnum y Bignum en Entero clase
  3. String admite asignaciones de mayúsculas y minúsculas Unicode, no solo ASCII
  4. Un nuevo método, Regexp # match ?, que es una versión booleana más rápida de Regexp # match
  5. La detección de interbloqueo de subprocesos ahora muestra subprocesos con su seguimiento y dependencia

La rama 2.4 también incluye mejoras de rendimiento para la tabla hash, Array # max, Array # min y el acceso a variables de instancia.

Ruby 2.5

Ruby 2.5.0 se lanzó el día de Navidad de 2017.

Algunos cambios notables incluyen:

  1. Las declaraciones de rescate y aseguramiento usan automáticamente un bloque do-end circundante (menos necesidad de bloques extra de inicio y fin )
  2. Encadenamiento de métodos con yield_self
  3. Soporte de cobertura de sucursales y medición de cobertura de método
  4. Transformaciones Hash más fáciles con Hash # slice y Hash # transform_keys

Además de eso, vienen muchas mejoras de rendimiento, como un paso de bloques más rápido (3 veces más rápido), Mutexes más rápidos, plantillas ERB más rápidas y mejoras en algunos métodos de concatenación.

Ruby 2.6

Ruby 2.6.0 se lanzó el día de Navidad de 2018.

Algunos cambios notables incluyen:

  1. JIT (experimental)
  2. RubyVM :: AbstractSyntaxTree (experimental)

Ruby 2.7

Ruby 2.7.0 se lanzó el día de Navidad de 2019.

Algunos cambios notables incluyen:

  1. Coincidencia de patrones (experimental)
  2. Mejora de REPL
  3. Compactación GC
  4. Separación de argumentos posicionales y de palabras clave

Ruby 3.0

Ruby 3.0.0 se lanzó el día de Navidad de 2020. Se conoce como Ruby 3x3. Uno de sus principales objetivos era cambiar el intérprete a un compilador Just-In-Time, para hacer que los programas fueran más rápidos.
